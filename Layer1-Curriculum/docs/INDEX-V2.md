## 🎓 How to Use This Documentation

### For Learners

**Week 0 (Preparation)**:
1. Read `layer1-plan-v2-modified.md` (full plan)
2. Skim `TEACHING-FRAMEWORK-V2.md` (understand approach)
3. Review `CHECKPOINT-GATES-V2.md` (know quality standards)
4. Set up environment (Python, Git, API keys)

**During Weeks 1-28**:
1. Follow weekly guide for current week
2. Complete scaffold files (TODOs)
3. Run tests and verify
4. Ship weekly artifact
5. Complete weekly reflection
6. Check checkpoint gates (Weeks 4, 8, 12, 16, 22)

**Resources to reference**:
- `CHECKPOINT-GATES-V2.md` - Before each gate
- `layer1-plan-v2-modified.md` Appendices - As needed
- `Extended-Plan.md` - For portfolio guidance

---

### For Curriculum Creators

**Before Creating Content**:
1. Read `TEACHING-FRAMEWORK-V2.md` (philosophy)
2. Read `WEEKLY-CONTENT-CREATION-GUIDE-V2.md` (templates)
3. Read `ACTION-FIRST-GUIDE.md` (action-first principles)
4. Read `WRITING-STYLE-GUIDE.md` (voice/tone)

**While Creating Content**:
1. Use templates from `WEEKLY-CONTENT-CREATION-GUIDE-V2.md`
2. Follow structure from `TEACHING-FRAMEWORK-V2.md`
3. Apply principles from `ACTION-FIRST-GUIDE.md`
4. Match voice/tone from `WRITING-STYLE-GUIDE.md`

**After Creating Content**:
1. Review against `QUALITY-CHECKLIST.md`
2. Verify checkpoint gate criteria (if applicable)
3. Test scaffold files (do they work?)
4. Get peer review
