# Curriculum Project Overview: The AI Engineer Accelerator

## 🚀 Unified Summary
The **AI Engineer Accelerator** is a comprehensive, multi-layered curriculum designed to transform software engineers into production-ready AI Engineers. Unlike standard "vibe-coding" tutorials, this repository implements a **Build-First Pedagogy** where technical rigor, engineering foundations, and guided discovery are the primary drivers of learning.

## 🏗️ Curriculum Layers

### 1. Layer 1: The Canonical Path (28 Weeks)
The authoritative roadmap for mastery, organized into five distinct phases:
- **Phase 1: Engineering Foundations** (Weeks 1-4) — Focusing on FastAPI, SQL, Testing (Pytest/Hypothesis), and Docker.
- **Phase 2: RAG Engineering** (Weeks 5-10) — From scratch implementation to advanced hybrid search and evaluation.
- **Phase 3: AI Agents** (Weeks 11-16) — ReAct patterns, LangGraph, Multi-Agent systems, and MCP.
- **Phase 4: Production Engineering** (Weeks 17-22) — Observability, Auth, Security, and Cloud deployment.
- **Phase 5: Capstone & Job Readiness** (Weeks 23-28) — Portfolio finalization and technical interview prep.

### 2. Layer 2: The Comprehensive Deep-Dive (73 Chapters)
A granular, technical encyclopedia (~102 hours) covering:
- **Advanced LLM Fundamentals**: Structured outputs, streaming, and error handling.
- **Specializations**: Domain-specific applications (e.g., Civil Engineering), Fine-Tuning, and GraphRAG.
- **Ecosystem Mastery**: Deep dives into LangChain Core, LlamaIndex, and Multi-Agent frameworks.

## 🎓 Pedagogical Framework & Methodologies

### 🧬 Core Philosophy: "Teach Them to Fish" 🎣
The curriculum is built on the principle that **Struggle IS the Learning**. It explicitly forbids AI-driven copy-pasting and instead enforces a **Guided Discovery** model.

### 🪜 The Teaching Ladder
AI Assistants supporting this curriculum must adhere to a 3-level escalation strategy:
1.  **Level 1: Conceptual Understanding** — Explain the *What* and *Why* using plain language and analogies before showing any code.
2.  **Level 2: Approach Suggestion** — Outline the steps and logic without providing the implementation.
3.  **Level 3: Pattern Examples** — Provide snippet-based guidance for specific technical blocks only when the student is stuck.

### 🛠️ Engineering-First Methodology
- **Anti-Vibe Coding**: Rejects the "just make it work" approach in favor of pydantic-driven data models and strictly typed systems.
- **Operational Rigor**: Mandatory inclusion of **Failure Logs**, **Cost Tracking**, and **Checkpoint Rubrics** for every project.
- **Flagship Evolution**: Projects are not one-offs; they evolve through versions (v1 → v2 → v3), teaching students how to refactor and scale AI systems.

## 🛠️ Technology Stack
- **Languages**: Python 3.10+ (Primary), Markdown (Curriculum).
- **Core Frameworks**: LangChain, LangGraph, LlamaIndex, CrewAI, AutoGen.
- **Infrastructure**: Docker, FastAPI, PostgreSQL (via pgvector), ChromaDB, FAISS.
- **Testing**: Pytest, Hypothesis (Property-based testing).
- **Providers**: OpenAI, Anthropic, Groq.
